2.2 Estructura global de un documento web

La estructura de un documento web se basa en varios componentes esenciales que trabajan en conjunto para crear una página funcional y accesible.

1. Declaración del tipo de documento (DOCTYPE)

La declaración <!DOCTYPE html> al inicio del documento indica al navegador que se está utilizando HTML5. Esto es crucial para asegurar que el navegador renderice la página de acuerdo con las normas de HTML5.

2. Elemento <html>

Este elemento es el contenedor principal de todo el contenido del documento. La etiqueta se puede acompañar con un atributo lang para especificar el idioma, como en <html lang="es">.

3. Elemento <head>

Incluye metadatos y enlaces a recursos externos, como hojas de estilo (CSS) y scripts de JavaScript. A continuación, se describen algunos elementos comunes dentro de <head>:

  • <meta charset="UTF-8">: Define la codificación de caracteres.
  • <meta name="viewport" content="width=device-width, initial-scale=1.0">: Controla el diseño responsivo en dispositivos móviles.
  • <link rel="stylesheet" href="estilo.css">: Vincula un archivo CSS externo para el estilo de la página.

4. Elemento <body>

Contiene el contenido visible de la página, como encabezados, párrafos, listas e imágenes. Aquí es donde se desarrolla la mayor parte de la estructura de la página web.

5. Uso de elementos semánticos

HTML5 introduce elementos semánticos que mejoran la accesibilidad y el SEO. Algunos de estos elementos son:

  • <header>: Representa el encabezado de la página o una sección.
  • <nav>: Define la sección de navegación.
  • <main>: Indica el contenido principal de la página.
  • <footer>: Representa el pie de página de un documento o sección.
  • <section>: Define secciones dentro del documento, cada una con su propio encabezado.
  • <article>: Contiene contenido independiente que puede distribuirse de forma autónoma.
  • <aside>: Representa contenido relacionado que no forma parte del contenido principal.

6. Ejemplo de Estructura Completa


<!DOCTYPE html>
<html lang="es">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Título de la Página</title>
    <link rel="stylesheet" href="estilo.css">
</head>
<body>
    <header>
        <h1>Encabezado Principal</h1>
        <nav>
            <ul>
                <li><a href="#seccion1">Sección 1</a></li>
                <li><a href="#seccion2">Sección 2</a></li>
            </ul>
        </nav>
    </header>
    <main>
        <section id="seccion1">
            <h2>Título de Sección 1</h2>
            <p>Contenido de la sección 1.</p>
        </section>
        <section id="seccion2">
            <h2>Título de Sección 2</h2>
            <p>Contenido de la sección 2.</p>
        </section>
    </main>
    <footer>
        <p>© 2024 Mi Página Web</p>
    </footer>
</body>
</html>

7. Buenas Prácticas

Al estructurar un documento HTML, es importante seguir buenas prácticas, como:

  • Utilizar etiquetas semánticas adecuadas para mejorar la accesibilidad.
  • Minimizar el uso de etiquetas <div> en favor de elementos semánticos.
  • Asegurarse de que el contenido sea accesible en dispositivos móviles mediante el uso de la meta etiqueta viewport.
  • Comprobar la validez del HTML utilizando herramientas de validación para evitar errores de marcado.
